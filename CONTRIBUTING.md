# Contributing

Contributions welcome! Here's how:

1. Fork the repo
2. Create a branch (`git checkout -b my-change`)
3. Make your changes
4. Test by installing the workflow in a test repo and triggering a scan
5. Commit using [conventional commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, etc.)
6. Open a PR

## What to contribute

- New scanner integrations
- Improved SARIF parsing / issue formatting
- Better deduplication logic
- Install script improvements (new platforms, package managers)
- Documentation fixes

## Reporting issues

Open a GitHub issue with:
- What you expected
- What happened
- Steps to reproduce (workflow logs help)
